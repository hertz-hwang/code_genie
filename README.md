## Quick Start

### 1. 准备输入文件

在项目根目录创建以下 5 个输入文件：

**input-fixed.txt** - 约束字根/字根组
```
# 单键位约束：所有字根固定到同一键位
传	a

# 多键位约束：字根组可在指定键位内移动
左 右	a d h
```

**input-roots.txt** - 动态字根组
```
禾 禾框 余字底
口 囗
```

**input-division.txt** - 汉字拆分表
```
明	日月	1000000
好	女子	800000
```

**pair_equivalence.txt** - 按键对当量
```
aq	0.8
wr	0.9
```

**key_distribution.txt** - 键位目标分布
```
a	3.5	1.0	1.0
s	3.5	1.0	1.0
```

### 2. 编译运行

```bash
# 构建（Release 模式）
cargo build --release

# 运行优化
cargo run --release 2>&1 | tee output.log
```

### 3. 查看结果

运行完成后，生成以下输出文件：

- `output-keymap.txt` - 字根键位映射表
- `output-encode.txt` - 汉字编码表  
- `output-distribution.txt` - 用指分布统计
- `output-equiv-dist.txt` - 当量分布统计

## 输入文件格式

### input-fixed.txt - 约束字根/字根组

**字根静态约束格式**：字根/字根组固定到某一键位上

```
传	a
根	o
```

**字根动态约束格式**：字根/字根组可在指定键位内移动

```
左 右	a d h
上 下	j k l
```

**格式说明**：
- `字根1 字根2 ... [tab] 键位(单个或空格分隔多个)`
- tab 分隔字根和键位列表
- 单个键位 → 字根固定到该键
- 多个键位 → 字根组可在这些键中选择

---

### input-roots.txt - 动态字根组

每行一组字根，组内字根共享同一编码（即归并字根），可分配到全局允许键位。

```
禾 禾框 余字底
口 囗
日 月
山 水
```

**说明**：
- 字根组共享同一键位编码
- 不同行的字根可以分配到不同键位
- 空行和以 `#` 开头的行为注释

---

### input-division.txt - 汉字拆分表

```
明	日月	1000000
好	女子	800000
国	玉口	500000
```

**格式说明**：
- `汉字 [tab] 字根1 字根2 ... [tab] 字频`
- 第 1 列：汉字（单个字符）
- 第 2 列：拆分后的字根，空格分隔
- 第 3 列：字频（整数，越大越常用）

**要求**：
- 所有汉字拆分表中的字根必须在 `input-fixed.txt` 或 `input-roots.txt` 中定义
- 缺失字根会导致程序报错并生成 `missing-roots.txt` 报告

---

### pair_equivalence.txt - 按键对当量

```
aq	0.8
wr	0.9
pl	1.2
```

**格式说明**：
- `按键对 [tab] 当量值`
- 当量值越小，表示这两个键连续输入越流畅

**默认行为**：文件不存在时自动使用全 0 当量表

---

### key_distribution.txt - 键位目标分布

```
a	3.5	1.0	1.0
s	3.5	1.0	1.0
d	4.0	1.2	0.8
```

**格式说明**：
- `键位 [tab] 目标频率% [tab] 低频惩罚 [tab] 高频惩罚`
- 目标频率：该键期望的用指占比
- 低频惩罚：实际频率低于目标时的惩罚系数（>1 鼓励使用）
- 高频惩罚：实际频率高于目标时的惩罚系数（<1 抑制过用）

**默认行为**：文件不存在时所有键目标频率为 0，不进行分布优化

## 配置参数

### 核心参数 (src/main.rs config 模块)

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `ALLOWED_KEYS` | `qwertyuopasdfghjklzxcbnm` | 允许使用的键位 |
| `NUM_THREADS` | `16` | 并行退火线程数（建议与 CPU 核心数相同） |
| `TOTAL_STEPS` | `100,000,000` | 每线程迭代步数 |
| `TEMP_START` | `1.0` | 初始温度（越高越容易跳出局部最优） |
| `TEMP_END` | `0.2` | 结束温度（越低收敛越好） |
| `DECAY_RATE` | `0.9998` | 温度衰减率（越接近 1 越慢冷却） |
| `SWAP_PROBABILITY` | `0.6` | 交换操作概率 |

---

### 目标函数权重

| 权重参数 | 默认值 | 优化目标 |
|----------|--------|----------|
| `WEIGHT_COLLISION_COUNT` | `0.5` | 重码数量（越小越好） |
| `WEIGHT_COLLISION_RATE` | `1.5` | 重码率（越小越好） |
| `WEIGHT_EQUIVALENCE` | `0.25` | 键位当量均值（越小越好） |
| `WEIGHT_EQUIV_CV` | `0.01` | 当量变异系数（越小越好，分布更均衡） |
| `WEIGHT_DISTRIBUTION` | `1.5` | 键位分布偏差（越小越好，用指更均衡） |

**调参建议**：
- 重码问题严重 → 提高 `WEIGHT_COLLISION_*`
- 打手感觉别扭 → 提高 `WEIGHT_EQUIVALENCE`
- 某些键过度使用 → 提高 `WEIGHT_DISTRIBUTION`

## 算法原理

### 模拟退火流程

```
1. 初始化：随机生成初始键位分配方案
        ↓
2. 迭代优化（1 亿步）
   ├─ 以 60% 概率：尝试交换两个字根组的键位
   └─ 以 40% 概率：尝试移动单个字根组到新键位
        ↓
3. 接受准则（Metropolis）
   ├─ 如果得分更低 → 接受新解
   └─ 如果得分更高 → 以概率 exp(-ΔE/T) 接受
        ↓
4. 自适应降温
   ├─ 接受率过低 → 升温（REHEAT_FACTOR = 1.5）
   ├─ 长期无改进 → 扰动跳出局部最优
   └─ 温度缓慢衰减（DECAY_RATE = 0.9998）
        ↓
5. 并行优化 16 次，选取得分最优结果
```

### 算法特性

1. **智能初始化**：按字频排序字根组，高频字优先分配空闲键位
2. **自适应降温**：根据接受率动态调整温度，避免过早收敛
3. **扰动机制**：每 50 万步进行批量扰动，跳出局部最优
4. **再加热策略**：停滞时临时升温 50%，恢复探索能力

## 常见问题

### Q: 如何检查字根是否完整？

运行程序，若提示缺失字根，查看 `missing-roots.txt` 文件补充定义。

### Q: 如何加快优化速度？

1. 减少 `TOTAL_STEPS`
2. 增加 `NUM_THREADS`（不超过 CPU 核心数）
3. 提高 `TEMP_END` 加快收敛
4. 使用 Release 模式编译

### Q: 如何提高优化质量？

1. 增加 `TOTAL_STEPS`（如 20000 万步）
2. 降低 `TEMP_END`（如 0.15）
3. 合理调整权重参数
4. 多次运行取最优结果

### Q: 为什么重码率总是降不下来？

- 检查 `WEIGHT_COLLISION_*` 权重是否足够高
- 增加迭代步数
- 检查字根定义是否充分（有些汉字无法避免重码）
